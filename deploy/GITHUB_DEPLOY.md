# Запуск бота 24/7 бесплатно через GitHub Actions

Бот будет сканировать рынок и слать сигналы **по расписанию (каждые ~15 минут)**
на серверах GitHub. **Компьютер выключен — сигналы всё равно приходят.** Бесплатно,
карта не нужна, VPN не нужен (серверы GitHub за границей).

> Режим отличается от «живого» бота: работают **автосигналы по таймеру**, а
> интерактивные команды (`/scan`, `/status`) и ликвидации в реальном времени — нет.

---

## Шаг 1. Создать аккаунт GitHub (бесплатно, без карты)

1. Открой https://github.com/signup
2. Введи почту, придумай пароль и имя пользователя, подтверди почту кодом.

## Шаг 2. Установить GitHub Desktop

1. Скачай: https://desktop.github.com/  → установи → войди своим аккаунтом GitHub.

## Шаг 3. Опубликовать проект

1. В GitHub Desktop: меню **File → Add local repository**.
2. Укажи папку: `C:\Users\Admin\Desktop\Cursor` → **Add repository**.
   - Всё уже подготовлено: посторонние файлы не попадут (настроен `.gitignore`),
     токен (`.env`) тоже не выгружается.
3. Нажми **Publish repository** (вверху).
   - Имя: например `crypto-signals`.
   - ⬜ **Сними** галочку **Keep this code private** (сделай репозиторий **публичным**).
     Почему: у публичных репозиториев GitHub Actions **бесплатны без лимита минут**,
     поэтому можно сканировать хоть каждые 15 минут. Код увидят другие, но **токен
     и получатели спрятаны в секретах** (в коде их нет) — это безопасно.
     *(Если всё же хочешь приватный — см. раздел «Лимиты» ниже, там нужно réже сканировать.)*
   - Нажми **Publish repository**.

Готово — код на GitHub.

## Шаг 4. Добавить секреты (токен и получатель)

На сайте github.com открой свой репозиторий → **Settings** (шестерёнка вверху) →
слева **Secrets and variables → Actions** → кнопка **New repository secret**.

Добавь **два** секрета (по очереди):

1. **Name:** `TELEGRAM_TOKEN`
   **Secret:** токен бота — скопируй его из файла `.env` (строка `TELEGRAM_TOKEN=...`,
   часть после `=`).

2. **Name:** `CHAT_IDS`
   **Secret:** `122245443`
   *(это твой Telegram chat id — уже известен из подписки).*

*(Необязательно)* если хочешь источник CoinMarketCap — добавь третий секрет
`COINMARKETCAP_API_KEY` со значением из `.env`.

## Шаг 5. Разрешить боту сохранять состояние

Там же в **Settings** → слева **Actions → General** → прокрути до
**Workflow permissions** → выбери **Read and write permissions** → **Save**.

*(Это нужно, чтобы бот запоминал, какие сигналы уже отправил, между запусками.)*

## Шаг 6. Включить и проверить

1. В репозитории вверху вкладка **Actions**.
2. Если попросит подтвердить запуск workflow — нажми **I understand… enable**.
3. Слева выбери workflow **crypto-signals** → кнопка **Run workflow** → **Run workflow**
   (ручной запуск для проверки).
4. Через 1–3 минуты задача выполнится (зелёная галочка). Если есть подходящие
   монеты — придёт сигнал в Telegram.

Дальше он будет запускаться **сам каждые ~15 минут**, без твоего участия.

---

## Важно: останови локального бота

Чтобы не приходили двойные сигналы, на этом компьютере больше **не запускай**
`python bot.py`. Если он сейчас запущен — скажи мне, я остановлю.

## Как поменять настройки позже

- Порог, число монет, cooldown, веса — в файле **`.env.ci`**. Поменял → в GitHub
  Desktop нажми **Commit** и **Push**. Изменения применятся со следующего запуска.
- Поменять частоту (например, каждые 10 минут) — в `.github/workflows/signals.yml`
  строка `cron: "*/15 * * * *"`.

## Возможные вопросы

- **Задержки:** GitHub иногда запускает по расписанию с задержкой в несколько минут —
  это нормально для бесплатного тарифа.
- **Лимиты:**
  - **Публичный репозиторий** (рекомендуется) — минуты Actions **безлимитны**, можно
    сканировать каждые 15 минут бесплатно.
  - **Приватный репозиторий** — только 2000 бесплатных минут в месяц. Каждые 15 мин
    их не хватит. Если хочешь приватный — открой `.github/workflows/signals.yml` и
    поставь запуск реже, например каждые 2 часа: `cron: "0 */2 * * *"` (тогда уложишься).
